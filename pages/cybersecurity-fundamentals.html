Cybersecurity Fundamentals

Core security concepts, frameworks, and methodologies essential for the Infrastructure & Cybersecurity Specialist role.

2.1 Core Security Concepts

What is Cybersecurity?

The practice of protecting systems, networks, and programs from digital attacks aimed at accessing, changing, or destroying sensitive information, extorting money from users, or interrupting normal business processes.

The CIA Triad

CIA TRIAD - Foundation of Information Security
                         
                          CONFIDENTIALITY 
                            (Secrecy)     
                         
                                  
               
                                                   
                                                   
                       
        INTEGRITY       AVAILABILITY   
       (Accuracy)                           (Access)      
                        

     CONFIDENTIALITY: Only authorized users can access data
     INTEGRITY: Data is accurate and unaltered  
     AVAILABILITY: Systems are accessible when needed
  • Confidentiality: Ensuring information is accessible only to authorized individuals. Implemented through encryption, access controls, and authentication.
  • Integrity: Ensuring data remains accurate and unmodified. Implemented through hashing, digital signatures, and checksums.
  • Availability: Ensuring systems and data are accessible when needed. Implemented through redundancy, backups, and disaster recovery.

Interview Questions

2.5 Interview Questions & Answers

Q1 Explain the NIST Cybersecurity Framework and how you would implement it at IBU.
Why They Ask

Tests understanding of industry frameworks and ability to apply them practically in a real-world educational environment.

Answer Framework
  • Overview of NIST CSF (5 core functions)
  • Current state assessment approach
  • Gap identification methodology
  • Implementation prioritization
  • Specific IBU application examples
Sample Answer

"The NIST Cybersecurity Framework consists of five core functions: Identify, Protect, Detect, Respond, and Recover. At IBU, I would implement it as follows:

Identify: First, I'd conduct an asset inventory of all IT systems, including M365 tenants, Meraki network infrastructure, 8x8 phone systems, and endpoint devices. I'd classify data based on sensitivity—student records, financial data, research materials.

Protect: Implement controls like MFA on all accounts via Azure AD, deploy Intune compliance policies for devices, ensure Meraki firewall rules follow least-privilege principles, and establish security awareness training for faculty, staff, and students.

Detect: Configure Microsoft Defender XDR for unified threat detection across endpoints, email, and cloud apps. Set up real-time alerting with defined thresholds to minimize false positives while catching genuine threats.

Respond: Develop incident response playbooks specific to IBU scenarios—ransomware affecting student systems, data breach of academic records, phishing targeting faculty. Define communication procedures with the CIO and IT Director.

Recover: Ensure robust backup systems with tested recovery procedures. Document RTO/RPO for critical systems like the student portal, email, and learning management systems."

Key Points to Hit
  • Demonstrate framework knowledge with all 5 functions
  • Show practical application with specific examples
  • Reference IBU-specific systems (M365, Meraki, 8x8)
  • Mention compliance requirements for education sector
Common Mistakes
  • Being too theoretical without practical examples
  • Not connecting to the specific role/organization
  • Forgetting to mention continuous improvement
Follow-up Questions
  • "How would you measure the maturity of each function?"
  • "What tools would you use to implement each function?"
  • "How would you prioritize if resources were limited?"
Q2 What is the difference between NIST and ISO 27001, and when would you use each?
Why They Ask

Tests depth of framework knowledge and ability to choose appropriate standards for different situations.

Sample Answer

"NIST CSF and ISO 27001 serve different purposes but can be complementary:

NIST CSF:

  • Voluntary framework focused on cybersecurity risk management
  • Organized around five functions (Identify, Protect, Detect, Respond, Recover)
  • Flexible, scalable to any organization size
  • Free to implement, commonly used in North America
  • Focused on cybersecurity specifically

ISO 27001:

  • International standard for Information Security Management Systems (ISMS)
  • Certification-based with formal audit requirements
  • Organized around 93 controls in 4 domains (Annex A)
  • Requires ongoing compliance audits
  • Broader scope including physical security, HR, legal

When to Use:

For IBU, I'd recommend using NIST CSF as the operational framework for day-to-day security operations because it's practical and widely understood. If IBU wanted formal certification to demonstrate security maturity to partners or for regulatory reasons, I'd map NIST controls to ISO 27001 and pursue certification.

In my previous role, we used NIST for internal security operations but maintained ISO 27001 certification for client-facing assurance."

Q3 Explain the MITRE ATT&CK framework and how you use it in threat detection.
Why They Ask

Demonstrates practical knowledge of threat intelligence and detection engineering - critical for the security monitoring aspects of this role.

Sample Answer

"MITRE ATT&CK is a knowledge base of adversary tactics, techniques, and procedures (TTPs) based on real-world observations. It's organized into a matrix with:

  • Tactics: The adversary's goals (Initial Access, Execution, Persistence, etc.)
  • Techniques: How they achieve those goals (Phishing, PowerShell, Registry Run Keys)
  • Sub-techniques: Specific variations of techniques

How I Use It:

1. Detection Engineering: I map our SIEM detection rules to ATT&CK techniques. For example, a rule detecting PowerShell downloading content maps to T1059.001 (PowerShell) and potentially T1105 (Ingress Tool Transfer).

2. Gap Analysis: I use the ATT&CK Navigator to visualize which techniques we can detect versus blind spots. This helps prioritize new detection development.

3. Threat Hunting: When hunting, I select specific techniques relevant to our threat landscape. For a university, I'd focus on techniques used in ransomware (T1486), credential theft (T1003), and data exfiltration (T1041).

4. Incident Investigation: During incidents, I classify attacker actions using ATT&CK to understand their progression and predict next steps.

For IBU, I'd ensure our Microsoft Defender XDR alerts are mapped to ATT&CK, creating a common language for the IT team when discussing threats."

Code Example - KQL Query Mapped to ATT&CK
KQL 
// T1059.001 - PowerShell - Detect encoded commands
DeviceProcessEvents
| where ProcessCommandLine contains "-EncodedCommand"
    or ProcessCommandLine contains "-enc"
    or ProcessCommandLine contains "FromBase64String"
| project TimeGenerated, DeviceName, AccountName, ProcessCommandLine
| order by TimeGenerated desc
Q4 How would you handle cybersecurity for a diverse user population (students, faculty, staff)?
Why They Ask

This directly addresses a key challenge of the role - balancing security with the needs of diverse user groups in an academic environment.

Sample Answer

"A university environment presents unique challenges because you have:

  • Students with varying technical expertise and personal devices
  • Faculty who prioritize academic freedom and may resist restrictions
  • Staff with access to sensitive administrative data
  • Visiting researchers and guests needing temporary access

My Approach:

1. Segmented Security Policies:

  • Students: Focus on awareness training, MFA enforcement, acceptable use policy
  • Faculty: Balance security with academic needs, provide secure research data handling
  • Staff: Stricter controls on financial and student data systems
  • Guests: Isolated network segment, time-limited access

2. User-Friendly Security:

  • Implement SSO to reduce password fatigue
  • Use risk-based authentication (allow low-risk logins, challenge high-risk)
  • Provide self-service password reset

3. Awareness Training:

  • Tailored training for each group (student-focused phishing examples)
  • Gamification to increase engagement
  • Regular simulated phishing tests

4. Technical Controls:

  • Conditional Access policies based on user risk, device compliance, location
  • DLP policies for sensitive data (student records, financial info)
  • Network segmentation (student WiFi vs. admin networks)

The key is balancing security with the academic mission—being too restrictive will push users to workarounds that create bigger risks."

Q5 What compliance frameworks are relevant to a Canadian university?
Why They Ask

Tests knowledge of Canadian privacy laws and education-specific regulations - critical for compliance at IBU.

Sample Answer

"For a Canadian university like IBU, several compliance frameworks apply:

Federal:

  • PIPEDA (Personal Information Protection and Electronic Documents Act): Governs how private-sector organizations collect, use, and disclose personal information. Applies to student and employee data.

Provincial (Ontario):

  • FIPPA (Freedom of Information and Protection of Privacy Act): Applies to Ontario universities regarding access to records and privacy protection.

International (if applicable):

  • GDPR: If IBU has students or operations in the EU, GDPR requirements for data protection apply.

Industry Standards:

  • PCI DSS: If processing student payments/tuition via credit card
  • NIST CSF: Best practice framework for cybersecurity operations

Education-Specific:

  • DLI (Designated Learning Institution) requirements for international student data
  • Ministry of Colleges and Universities reporting requirements

My Implementation Approach:

  1. Map all compliance requirements to a unified control framework
  2. Identify overlapping controls to avoid duplicate effort
  3. Implement technical controls (encryption, access controls, audit logging)
  4. Document policies and procedures
  5. Conduct regular audits and gap assessments
  6. Maintain evidence of compliance for auditors
