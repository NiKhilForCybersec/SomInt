Interview Questions

8.2 Interview Questions

Q1What experience do you have with VoIP/UCaaS administration?
Sample Answer

"I have experience administering VoIP systems including:"

  • User Management: Provisioning extensions, managing hunt groups, configuring auto-attendants
  • Call Routing: Setting up ring groups for departments, after-hours routing, holiday schedules
  • Integration: SSO with Azure AD for single sign-on, directory sync for contacts
  • Security: Ensuring encryption is enforced, setting call restrictions for international dialing
  • Troubleshooting: Call quality issues (QoS configuration), connectivity problems, softphone issues

For IBU's 8x8 system, I'd integrate with Azure AD for SSO and ensure proper call routing for different departments - admissions, registrar, IT help desk, etc."

Q2How would you secure a VoIP system?
Sample Answer

"VoIP security requires attention to both infrastructure and administrative controls:"

Network Security:

  • Dedicated VLAN for voice traffic - isolate from data network
  • QoS prioritization for voice packets
  • Firewall rules limiting SIP/RTP traffic to known endpoints
  • NAT traversal considerations for remote workers

Encryption:

  • TLS for signaling (SIP over TLS)
  • SRTP for media encryption
  • Disable unencrypted fallback
  • Verify encryption is end-to-end, not just to provider

Access Control:

  • SSO integration (Azure AD)
  • MFA for admin access
  • Role-based permissions (admin vs. user vs. receptionist)
  • Separate admin credentials from user accounts

Fraud Prevention:

  • International call restrictions by default
  • Premium rate number blocking
  • Anomaly detection for unusual calling patterns
  • Regular audit of call records and billing
  • PIN protection for outbound long distance
Q3A professor reports terrible call quality during lectures. How do you troubleshoot?
Sample Answer

"Call quality issues are usually network-related. My systematic approach:"

Gather Information:

  • When does it happen? (specific times = congestion)
  • Consistent or intermittent?
  • Wired or WiFi connection?
  • What symptoms? (choppy, delayed, one-way audio, drops)

Diagnose by Symptom:

  • Choppy/robotic audio: Likely packet loss - check network path, WiFi interference
  • Delayed audio: High latency - check WAN links, QoS misconfiguration
  • One-way audio: NAT/firewall issue - check SIP ALG settings, port forwarding
  • Complete drops: Network instability - check switch/AP logs for errors

Technical Investigation:

  • Check 8x8 analytics dashboard for call quality scores (MOS)
  • Review Meraki network analytics during problem timeframe
  • Verify QoS is properly tagging voice traffic (DSCP 46 for voice)
  • If WiFi: check for channel interference, consider wired connection
  • Test from different location to isolate local vs. network-wide issue

Common Fixes:

  • Enable QoS if not configured
  • Move to wired connection if WiFi-related
  • Increase bandwidth allocation for voice VLAN
  • Update network drivers on endpoint
  • Disable SIP ALG on router if causing NAT issues
Q4What is toll fraud and how do you prevent it?
Sample Answer

"Toll fraud is when attackers gain unauthorized access to your phone system to make expensive calls - usually international or premium rate numbers - leaving you with the bill."

How It Happens:

  • Compromised credentials: Attackers guess weak voicemail PINs or admin passwords
  • DISA abuse: Direct Inward System Access lets external callers dial out - attackers exploit this
  • SIP registration hijacking: Attackers register unauthorized devices
  • After-hours attacks: Most fraud occurs nights/weekends when no one is watching

Prevention Measures:

  • Strong authentication: Complex voicemail PINs, MFA for admin access
  • International call restrictions: Whitelist specific countries needed, block all others by default
  • Premium rate blocking: Block 900 numbers and international premium services
  • Call spending limits: Set alerts and auto-cutoffs for unusual spending
  • Disable unused features: Turn off DISA, call forwarding to external if not needed

Detection & Response:

  • Monitor call detail records for unusual patterns
  • Set alerts for international calls outside business hours
  • Review monthly bills for unexpected charges
  • If fraud detected: disable affected extensions immediately, contact carrier

One weekend of toll fraud can result in tens of thousands of dollars in charges. Prevention is far cheaper than cleanup."

Q5What's the difference between UCaaS and traditional PBX?
Sample Answer

"Understanding this is important for planning and troubleshooting:

Traditional PBX (Private Branch Exchange):

  • Location: Physical hardware on-premises
  • Ownership: You buy, maintain, and upgrade it
  • Connections: Analog lines (POTS) or digital trunks (PRI/T1)
  • Features: Fixed - what you bought is what you get
  • Scaling: Requires hardware purchase and installation
  • Cost model: High upfront, lower ongoing (but maintenance/upgrades add up)

UCaaS (Unified Communications as a Service) - like 8x8:

  • Location: Cloud-hosted by provider
  • Ownership: Subscription service - provider manages everything
  • Connections: Internet-based (SIP trunking)
  • Features: Constantly updated - video, chat, SMS, integrations
  • Scaling: Add licenses instantly - no hardware needed
  • Cost model: Low upfront, predictable monthly per-user

Why Universities Often Choose UCaaS:

  • No need for specialized telephony staff
  • Easy integration with Teams/M365
  • Supports remote/hybrid work seamlessly
  • Automatic disaster recovery - if campus goes down, calls still work
  • Predictable budgeting (OpEx vs. CapEx)

The tradeoff is dependency on internet connectivity and the provider's reliability."
