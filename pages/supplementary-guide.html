1. Polycom VoIP Phones

What You Need to Know

IBU uses Polycom phones with 8x8. Polycom is a leading VoIP hardware manufacturer (now part of HP/Poly).

Common Polycom Models

ModelUse CaseKey Features
VVX 150/250Basic desk phone2-4 lines, HD audio
VVX 350/450Standard office6-12 lines, color display, Bluetooth
CCX 400/500/600Teams-native phonesBuilt for Microsoft Teams
Trio 8500/8800Conference rooms360° pickup, daisy-chain capable

Provisioning with 8x8

  1. Factory reset the phone (if previously used)
  2. Connect to network - DHCP assigns IP, phone gets provisioning URL
  3. 8x8 Admin Console → Phones → Add Device → Enter MAC address
  4. Assign to user - Phone auto-downloads config from 8x8 cloud
  5. Test - Verify dial tone, inbound/outbound calls, voicemail

Common Troubleshooting

IssueCauseSolution
No dial toneRegistration failedCheck network, verify MAC in 8x8
One-way audioNAT/firewall issueCheck SIP ALG settings, port forwarding
Echo/feedbackAudio settingsAdjust sidetone, check handset
Can't transfer callsFeature not enabledCheck user permissions in 8x8 admin
Q1 A faculty member says their Polycom phone isn't ringing. How do you troubleshoot?
Sample Answer

"I'd take a systematic approach:

  1. Check physical: Is the phone powered on? Check Ethernet cable, PoE switch port
  2. Check registration: Does the phone display show 'Registered' or an error?
  3. Check 8x8 Admin: Is the phone showing online? Is it assigned correctly?
  4. Check DND: Is Do Not Disturb enabled on phone or in 8x8 settings?
  5. Check call forwarding: Are calls being forwarded elsewhere?
  6. Test: Call from my cell to isolate if it's inbound-only or all calls

Usually it's either DND accidentally enabled or a network registration issue."

2. Zoom Enterprise Administration

Zoom Admin Portal Key Areas

SectionWhat You Manage
User ManagementAdd/remove users, assign licenses, set roles
Account SettingsDefault meeting settings, security, recording policies
Room ManagementZoom Rooms for conference spaces
ReportsUsage analytics, meeting history

Critical Security Settings for a University

  • Waiting rooms: Enable by default - prevents Zoom bombing
  • Passcodes: Require for all meetings
  • Authentication: Require sign-in for external participants
  • Screen sharing: Host only by default
  • Recording: Disable local, enable cloud with retention policy
  • SSO: Integrate with Azure AD for single sign-on
Q2 How would you prevent Zoom bombing for faculty lectures?
Sample Answer

"Multiple layers of protection:

  1. Waiting rooms: All attendees wait until host admits them
  2. Passcodes: Required, shared only through LMS (not public posts)
  3. Authentication: Require participants to be signed into their IBU account
  4. Lock meeting: Train faculty to lock once class starts
  5. Disable join before host: Prevents early troublemakers
  6. Screen sharing host-only: Students can't hijack the screen

I'd also provide faculty training and a quick-reference guide."

3. KQL Query Examples for Microsoft Sentinel

Essential KQL Queries

1. Failed Sign-in Attempts (Brute Force Detection)

SigninLogs
| where ResultType != 0
| where TimeGenerated > ago(24h)
| summarize FailedAttempts = count() by UserPrincipalName, IPAddress
| where FailedAttempts > 10
| order by FailedAttempts desc

2. Malware Detected on Endpoints

DeviceEvents
| where ActionType == "AntivirusDetection"
| where TimeGenerated > ago(24h)
| project TimeGenerated, DeviceName, FileName, FolderPath, SHA256
| order by TimeGenerated desc

3. New Admin Role Assignments

AuditLogs
| where OperationName == "Add member to role"
| where TimeGenerated > ago(7d)
| extend RoleName = tostring(TargetResources[0].displayName)
| project TimeGenerated, RoleName, InitiatedBy

4. Large File Downloads (Data Exfiltration)

OfficeActivity
| where Operation == "FileDownloaded"
| where TimeGenerated > ago(24h)
| summarize DownloadCount = count(), TotalSize = sum(FileSize) by UserId
| where DownloadCount > 100 or TotalSize > 1000000000
| order by TotalSize desc

4. PowerShell Commands to Memorize

M365 User Management

# Connect to M365
Connect-MgGraph -Scopes "User.ReadWrite.All"

# Get user info
Get-MgUser -UserId "user@ibu.ca"

# Reset password
$password = @{ Password = "TempP@ss123!"; ForceChangePasswordNextSignIn = $true }
Update-MgUser -UserId "user@ibu.ca" -PasswordProfile $password

# Block sign-in
Update-MgUser -UserId "user@ibu.ca" -AccountEnabled:$false

Exchange Online

# Connect
Connect-ExchangeOnline

# Check mailbox size
Get-MailboxStatistics -Identity "user@ibu.ca" | Select TotalItemSize

# Convert to shared mailbox (departing employee)
Set-Mailbox -Identity "departed@ibu.ca" -Type Shared

# Search for phishing emails
Get-MessageTrace -SenderAddress "attacker@evil.com" -StartDate (Get-Date).AddDays(-7)

5. Common Helpdesk Scenarios

Scenario: "My computer is slow"

  1. Check Task Manager - what's using CPU/RAM?
  2. Check for pending Windows updates or restarts
  3. Check disk space (C: drive full?)
  4. Check for malware via Defender scan
  5. Check startup programs - disable unnecessary ones

Scenario: "I can't connect to WiFi"

  1. Verify correct SSID (Staff vs Guest)
  2. Check if other users affected (network vs device)
  3. Forget and reconnect to network
  4. Check Meraki dashboard for AP status
  5. Verify 802.1X credentials

Scenario: "I think I clicked a phishing link"

  1. Don't panic - thank them for reporting!
  2. Ask: Did you enter credentials? Download anything?
  3. If credentials entered: Force password reset immediately
  4. If download: Disconnect from network, run full scan
  5. Check Azure AD sign-in logs for suspicious activity
