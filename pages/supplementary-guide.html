1. Polycom VoIP Phones (HIGH PRIORITY)

What You Need to Know

IBU uses Polycom phones with 8x8. Polycom is a leading VoIP hardware manufacturer (now part of HP/Poly).

Provisioning Polycom Phones with 8x8

  1. Factory reset the phone (if previously used)
  2. Connect to network - DHCP assigns IP, phone gets provisioning URL
  3. 8x8 Admin Console ‚Üí Phones ‚Üí Add Device ‚Üí Enter MAC address
  4. Assign to user - Phone auto-downloads config from 8x8 cloud
  5. Test - Verify dial tone, inbound/outbound calls, voicemail

IssueCauseSolution

Interview Question & Answer

Q: "A faculty member says their Polycom phone isn't ringing. How do you troubleshoot?"

"I'd take a systematic approach:

  1. Check physical: Is the phone powered on? Check Ethernet cable, PoE switch port
  2. Check registration: Does the phone display show 'Registered' or an error? Look at the phone's status screen
  3. Check 8x8 Admin: Is the phone showing online in the admin console? Is it assigned to the correct user?
  4. Check DND: Is Do Not Disturb enabled? Check both on the phone and in 8x8 user settings
  5. Check call forwarding: Are calls being forwarded elsewhere?
  6. Test: Call from my cell to isolate if it's inbound-only or all calls

Usually it's either DND accidentally enabled or a network registration issue."

2. Zoom Enterprise Administration (HIGH PRIORITY)

Zoom Admin Portal Key Areas

SectionWhat You Manage
User ManagementAdd/remove users, assign licenses, set roles
Account SettingsDefault meeting settings, security, recording policies
Room ManagementZoom Rooms for conference spaces
ReportsUsage analytics, meeting history, registration
IM ManagementZoom Chat settings and policies

Critical Security Settings for a University

  • Waiting rooms: Enable by default - prevents Zoom bombing
  • Passcodes: Require for all meetings
  • Authentication: Require sign-in for external participants
  • Screen sharing: Host only by default
  • Recording: Disable local recording, enable cloud with retention policy
  • Chat: Disable file transfer in meetings
  • SSO: Integrate with Azure AD for single sign-on

Zoom SSO Integration with Azure AD

  1. In Zoom Admin ‚Üí Single Sign-On ‚Üí Enable SAML
  2. In Azure AD ‚Üí Enterprise Apps ‚Üí Add Zoom
  3. Configure SAML: Copy metadata URLs between platforms
  4. Map attributes: email, displayName, department
  5. Test with a pilot user before enforcing

Interview Question & Answer

Q: "How would you prevent Zoom bombing for faculty lectures?"

"Multiple layers of protection:

  1. Waiting rooms: All attendees wait until host admits them
  2. Passcodes: Required, shared only through LMS (not public posts)
  3. Authentication: Require participants to be signed into their IBU account
  4. Lock meeting: Train faculty to lock once class starts
  5. Disable join before host: Prevents early troublemakers
  6. Screen sharing host-only: Students can't hijack the screen

I'd also provide faculty training and a quick-reference guide for removing disruptive participants."

Teams Rooms Basics

Teams Rooms are dedicated meeting room devices running a specialized Teams client. They typically include:

  • Touch controller: Tablet for starting/controlling meetings

  • Display: TV or projector

Teams Rooms Administration

4. KQL Query Examples for Microsoft Sentinel (MEDIUM PRIORITY)

Essential KQL Queries You Should Know

1. Failed Sign-in Attempts (Brute Force Detection)

SigninLogs
| where ResultType != 0  // Non-successful logins
| where TimeGenerated > ago(24h)
| summarize FailedAttempts = count() by UserPrincipalName, IPAddress
| where FailedAttempts > 10
| order by FailedAttempts desc

2. Impossible Travel Detection

SigninLogs
| where TimeGenerated > ago(7d)
| where ResultType == 0  // Successful logins only
| summarize Locations = make_set(Location), 
            LoginCount = count() by UserPrincipalName
| where array_length(Locations) > 3

3. Malware Detected on Endpoints

DeviceEvents
| where ActionType == "AntivirusDetection"
| where TimeGenerated > ago(24h)
| project TimeGenerated, DeviceName, FileName, 
          FolderPath, SHA256, AccountName
| order by TimeGenerated desc

4. New Admin Role Assignments

AuditLogs
| where OperationName == "Add member to role"
| where TimeGenerated > ago(7d)
| extend RoleName = tostring(TargetResources[0].displayName)
| extend AddedUser = tostring(TargetResources[0].userPrincipalName)
| project TimeGenerated, RoleName, AddedUser, InitiatedBy

5. Data Exfiltration - Large Downloads

OfficeActivity
| where Operation == "FileDownloaded"
| where TimeGenerated > ago(24h)
| summarize DownloadCount = count(), 
            TotalSize = sum(FileSize) by UserId
| where DownloadCount > 100 or TotalSize > 1000000000
| order by TotalSize desc

Interview Question & Answer

Q: "How would you use KQL to detect a compromised account?"

"I'd look for several indicators:

  1. Impossible travel: Logins from geographically distant locations within short time
  2. Failed then success: Multiple failures followed by success = password guessed
  3. Unusual timing: Logins at 3 AM for a user who normally works 9-5
  4. New devices/apps: First-time access from unknown device or legacy auth
  5. Inbox rules: New forwarding rules (attackers hide their tracks)

I'd create a hunting query combining these signals and potentially build an analytics rule to alert automatically."

5. eFax Administration (MEDIUM PRIORITY)

What is eFax?

eFax is a cloud fax service that sends/receives faxes via email or web portal - no physical fax machine needed.

Why Universities Still Use Fax

  • Registrar: Transcript requests from other institutions
  • Financial Aid: Government forms, verification documents
  • HR: Background check authorizations
  • Legal: Some documents still require fax for compliance

eFax Admin Tasks

  • Assign fax numbers to users/departments
  • Configure email delivery (faxes arrive as PDF attachments)
  • Set retention policies for fax history
  • Manage cover page templates

6. Onsite IT Support Scenarios (MEDIUM PRIORITY)

Common Support Tickets You'll Handle

The job mentions "Onsite IT Support" - expect questions about day-to-day troubleshooting:

Scenario 1: "My computer is slow"

  1. Check Task Manager - what's using CPU/RAM?
  2. Check for pending Windows updates or restarts
  3. Check disk space (C: drive full?)
  4. Check for malware via Defender scan
  5. Check startup programs - disable unnecessary ones
  6. If old hardware: recommend replacement or RAM upgrade

Scenario 2: "I can't connect to WiFi"

  1. Verify they're connecting to correct SSID (Staff vs Guest)
  2. Check if other users in same area are affected
  3. Forget and reconnect to network
  4. Check Meraki dashboard for AP status
  5. Verify 802.1X credentials if enterprise auth
  6. Try different device to isolate user device vs network issue

Scenario 3: "Outlook won't open / is frozen"

  1. Close and reopen Outlook
  2. Try Outlook in Safe Mode: outlook.exe /safe
  3. Check for Office updates
  4. Disable add-ins one by one
  5. Repair Office installation
  6. Delete and recreate Outlook profile (last resort)

Scenario 4: "I think I clicked a phishing link"

  1. Don't panic - thank them for reporting!
  2. Ask: Did you enter any credentials? Download anything?
  3. If credentials entered: Force password reset immediately
  4. If download: Disconnect from network, run full scan
  5. Check Azure AD sign-in logs for suspicious activity
  6. Report the phishing email to security team
  7. Document the incident

7. Quick Reference Commands

PowerShell Commands to Memorize

M365 User Management

# Connect to M365
Connect-MgGraph -Scopes "User.ReadWrite.All"

# Get user info
Get-MgUser -UserId "user@ibu.ca"

# Reset password
$password = @{ Password = "TempP@ss123!"; ForceChangePasswordNextSignIn = $true }
Update-MgUser -UserId "user@ibu.ca" -PasswordProfile $password

# Check license
Get-MgUserLicenseDetail -UserId "user@ibu.ca"

# Block sign-in
Update-MgUser -UserId "user@ibu.ca" -AccountEnabled:$false

Exchange Online

# Connect
Connect-ExchangeOnline

# Check mailbox size
Get-MailboxStatistics -Identity "user@ibu.ca" | Select TotalItemSize

# Add full access to another mailbox
Add-MailboxPermission -Identity "shared@ibu.ca" -User "user@ibu.ca" -AccessRights FullAccess

# Convert to shared mailbox (for departing employee)
Set-Mailbox -Identity "departed@ibu.ca" -Type Shared

# Search for phishing emails
Get-MessageTrace -SenderAddress "attacker@evil.com" -StartDate (Get-Date).AddDays(-7) -EndDate (Get-Date)

Intune/Endpoint

# Force device sync
Invoke-IntuneManagedDeviceSyncDevice -managedDeviceId "device-id"

# Get device compliance status
Get-MgDeviceManagementManagedDevice | Where-Object {$_.ComplianceState -ne "compliant"}
