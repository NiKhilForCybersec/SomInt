6.2 Interview Questions

Q1 How would you design network segmentation for a university campus?
â–¼
Sample Answer

"I would implement VLAN-based segmentation using Meraki:

  • Management VLAN: Network devices, highly restricted
  • Staff VLAN: Admin staff with access to sensitive systems
  • Faculty VLAN: Academic staff, research systems
  • Student VLAN: Labs, wireless for student devices
  • Guest VLAN: Isolated, internet-only access
  • IoT VLAN: Printers, cameras, building systems - restricted internet

Firewall rules between VLANs enforce least-privilege access. Guest and IoT networks cannot reach internal resources."

Q2 Explain your experience with Cisco Meraki and its advantages.
â–¼
Sample Answer

"Meraki's cloud-managed approach offers several advantages for a university:

  • Single Dashboard: Manage all network devices from one interface
  • Zero-Touch Deployment: Pre-configure devices, they auto-configure on-site
  • Automatic Updates: Firmware updates scheduled and automated
  • Built-in Security: IDS/IPS, content filtering, malware protection
  • Location Analytics: Track foot traffic, useful for campus planning
  • API Integration: Automate with scripts, integrate with SIEM

For IBU, I'd leverage Meraki's group policies to automatically apply appropriate restrictions based on whether a device is staff, student, or guest."

Q4 What is 802.1X and how does it improve network security?
â–¼
Sample Answer

"802.1X is a port-based network access control standard that authenticates devices before granting network access:

How It Works:

  • Supplicant: The device requesting access (laptop, phone)
  • Authenticator: The switch or access point
  • Authentication Server: RADIUS server (like Azure AD with NPS)

Process Flow:

  1. Device connects to network port or SSID
  2. Switch/AP blocks all traffic except authentication
  3. Device presents credentials (certificate, username/password)
  4. RADIUS validates against directory (Azure AD)
  5. If valid, port opens and assigns appropriate VLAN

Security Benefits:

  • No network access without valid credentials
  • Dynamic VLAN assignment based on user/device
  • Prevents rogue devices on wired network
  • Audit trail of who connected where and when
  • Can enforce device compliance (certificate = managed device)

For a university, 802.1X ensures only authenticated students and staff get campus network access."

Q5 How would you troubleshoot slow network performance in a specific building?
â–¼
Sample Answer

"Systematic troubleshooting approach:

1. Gather Information:

  • When did it start? (time-based = scheduled process or congestion)
  • Wired or WiFi? (different troubleshooting paths)
  • All users or specific ones? (user device vs. infrastructure)
  • All applications or specific ones? (app-specific vs. network)

2. Check Meraki Dashboard:

  • Switch/AP health - any devices showing errors?
  • Client count - is the building overloaded?
  • Bandwidth utilization - uplink saturated?
  • Latency graphs - when did performance degrade?
  • Top talkers - is one device consuming all bandwidth?

3. Common Causes & Fixes:

  • WiFi interference: Check channel utilization, adjust channels
  • Uplink saturation: Identify bandwidth hog, implement QoS
  • Spanning tree issues: Check for topology changes, loops
  • Failing hardware: Check error counters, replace if needed
  • DNS issues: Slow DNS = slow everything, test resolution

4. Escalation Path:

  • If ISP issue: Contact provider with evidence
  • If Meraki issue: Open support case with dashboard data

The key is using the dashboard data to narrow down the problem before physically investigating."

Q6 What is QoS and why is it important in a university network?
â–¼
Sample Answer

"QoS (Quality of Service) prioritizes certain types of network traffic over others:

Why It Matters:

  • Not all traffic is equal - voice calls need low latency, downloads can wait
  • During congestion, critical apps maintain performance
  • Prevents bandwidth hogs from impacting everyone

University QoS Priorities:

PriorityTraffic TypeDSCP Value
1 (Highest)Voice (VoIP, Teams calls)46 (EF)
2Video conferencing34 (AF41)
3Interactive apps (LMS, SIS)26 (AF31)
4Business apps, email18 (AF21)
5 (Lowest)Bulk downloads, streaming0 (BE)

Implementation:

  • Tag traffic at source (endpoints, applications)
  • Honor tags at switches and APs
  • Configure WAN interface to prioritize during congestion
  • Monitor to ensure policies are working

During registration week when bandwidth spikes, QoS ensures faculty Teams meetings still work while students stream Netflix."
