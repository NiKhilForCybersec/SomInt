Days 1-30: Learn

15.2 Days 1-30: Learn & Understand

First 30 Days - Detailed Activities

Week 1: Orientation & Relationships

DayFocus AreaSpecific Activities
1-2 HR & Admin Complete onboarding paperwork, get badge/access, set up workstation, meet IT team
3-4 Manager Alignment 1:1 with IT Director: understand expectations, priorities, pain points, political landscape
5 Stakeholder Introductions Meet key contacts: Registrar, Finance, HR, Faculty representatives

Week 2-3: Technical Discovery

  • Document current architecture: Network topology, server inventory, cloud services, security tools
  • Review existing policies: Acceptable use, security policies, incident response procedures
  • Audit access: Who has admin rights? Are they documented and appropriate?
  • Examine logs: What are we monitoring? What alerts exist? What's noise vs. signal?
  • Test backup/recovery: Verify backups are working and restoration is documented

Week 4: Risk Assessment

  • Compile findings into a baseline security assessment
  • Identify top 5 risks based on likelihood and impact
  • Note quick wins that can be achieved with minimal effort/cost
  • Draft preliminary observations for manager discussion

Days 31-60: Contribute

15.3 Days 31-60: Contribute & Build

Second 30 Days - Taking Ownership

Operational Ownership:

  • Take primary responsibility for security monitoring and alert triage
  • Handle endpoint security incidents independently
  • Manage vulnerability scanning and patch coordination
  • Support user access requests and M365 administration

Quick Wins to Target:

Quick WinEffortImpactVisibility
Enable MFA for remaining users Low High Medium
Clean up stale AD accounts Low Medium Low
Implement email authentication (DMARC) Medium High Low
Create new employee security checklist Low Medium High
Launch first phishing simulation Medium High High
Document incident response procedures Medium High Medium

Process Improvements:

  • Create/update SOPs for common tasks (user provisioning, device setup, incident response)
  • Establish regular security check-in with IT Director (weekly initially)
  • Set up basic security metrics dashboard
  • Begin vendor relationship management (understand contracts, support processes)

Day 60 Deliverable: Quick Wins Report
Document completed improvements with before/after metrics where possible. Example: "MFA enrollment increased from 65% to 98%", "Reduced stale accounts from 47 to 0", "First phishing simulation completed: 18% click rate, establishing baseline."
Days 61-90: Lead

15.4 Days 61-90: Lead & Strategize

Final 30 Days - Strategic Leadership

Strategic Initiatives:

  • Security Roadmap: Develop 12-month security improvement plan with quarterly milestones
  • Budget Proposal: Identify investments needed, build business cases with ROI
  • Risk Register: Formalize risk documentation with ownership and treatment plans
  • Training Program: Launch security awareness program for all staff

Relationship Expansion:

  • Present security overview to leadership team
  • Establish regular touchpoints with key department heads
  • Build relationships with external partners (vendors, peer institutions)
  • Consider joining higher-ed security communities (EDUCAUSE, REN-ISAC)

Continuous Improvement:

  • Conduct tabletop exercise with IT team (incident response practice)
  • Review and update policies based on first 60 days of observations
  • Identify automation opportunities to improve efficiency
  • Plan for next quarter's priorities
